# UI/UX Design Skill

## When To Use

- Designing web pages, dashboards, components, and Discord response layouts.
- Improving visual hierarchy, spacing, color, typography, and interaction states.
- Auditing accessibility, contrast, and responsive behavior.

## Design Principles

- Start from user intent and information priority.
- Make primary actions visually clear.
- Use consistent spacing and type scale.
- Keep color meaning stable across the product.
- Do not rely on color alone to communicate state.
- Prefer useful motion over decorative noise.

## Discord Embed Guidance

- Keep titles short and action-oriented.
- Use descriptions for primary information.
- Use fields for secondary structured data.
- Use timestamps on action/history embeds.
- Use thumbnails for user or server identity.
- Keep footer text consistent across command families.

## Website Guidance

- Mobile-first responsive layout.
- Semantic landmarks: `header`, `nav`, `main`, `section`, `footer`.
- Accessible labels for icon-only controls.
- Visible focus styles for keyboard users.
- Body text contrast should meet accessibility requirements.
- Use existing design tokens before inventing new ones.

## Visual System Checklist

- Palette has a clear primary, success, warning, danger, and muted state.
- Typography has distinct heading/body/caption roles.
- Components have hover, focus, active, disabled, loading, and error states.
- Empty states explain what happened and what to do next.
- Animations are short and purposeful.
