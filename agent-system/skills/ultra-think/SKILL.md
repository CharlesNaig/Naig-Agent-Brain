# Ultra Think Skill

## When To Use

- The task has multiple systems, phases, or unknowns.
- The user asks for deep planning, architecture, risk analysis, or phased decomposition.
- Implementation should not start yet.

## Process

1. Clarify goal, scope, constraints, stakeholders, and unknowns.
2. Decompose by layer, component, user journey, or research unknown.
3. Identify risks, assumptions, and decisions.
4. Produce a phased execution plan.
5. Define done, verification, and out-of-scope work.
6. End with the recommended first action.

## Output Shape

```markdown
# UltraPlan: Title

## Summary

## Assumptions

## Risk And Decision Map

## Architecture Overview

## Phased Execution Plan

## Definition Of Done

## Out Of Scope

## Recommended First Action
```

## Rules

- Do not start coding during an ultra-think session unless explicitly asked.
- Be explicit about assumptions.
- Prefer practical phases over oversized plans.
- Surface tradeoffs and risks clearly.
